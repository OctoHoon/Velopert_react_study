React : 2013 년 release 1장: 리액트 입문 01. 리액트는 어쩌다 만들어졌을까? JS로 UI을 제어하려면 DOM Selector API를 사용해 특정 DOM 선택한 뒤, 이벤트에 따른 변화를 설정해야 함 DOM(Document object Model) 규모가 커지면 이벤트 핸들러와 DOM 업데이트의 관계가 매우 복잡해짐 리액트는 어떠한 상태가 바뀌었을때, 그 상태에 따라 DOM 을 어떻게 업데이트 할 지 규칙을 정하는 것이 아니라, 아예 다 날려버리고 처음부터 모든걸 새로 만들어서 보여즘 이 때, 속도 문제를 해결하기 위해 "Virtual DOM" 도입 메모리에 가상으로 존재하는 DOM 작동 성능이 실제로 브라우저에서 DOM 을 보여주는 것 보다 속도가 훨씬 빠릅 리액트는 상태가 업데이트 되면, 업데이트가 필요한 곳의 UI 를 Virtual DOM 을 통해서 렌더링합니다. 그리고 나서 리액트 개발팀이 만든 매우 효율적인 비교 알고리즘을 통하여 실제 브라우저에 보여지고 있는 DOM 과 비교를 한 후, 차이가 있는 곳을 감지하여 이를 실제 DOM 에 패치 02. 작업환경 준비 - Node.js - Yarn - 코드 에디터 (Atom, WebStorm 등) - Git bash 03. 리액트 컴포넌트 생성 04.
